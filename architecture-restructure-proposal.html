Implemented architecture: a publication-shaped ESPI pipeline

ESPI now uses four readable scientific phase scripts and a minimal R package. The analysis, frozen objects, figure identities, tables, and rendered notebook remain the publication contract; framework-only machinery has been removed.

1. Non-negotiable invariants

Scientific invariants

  • Counts processing, QC rules, PFlog/log1p transformations, HVG selection, PCA, UMAP, Leiden grids, MG selection, marker testing, pseudobulk DESeq2 models, shrinkage, and enrichment methods do not change.
  • Final source object remains PFlog with cell-cycle genes eligible as HVGs, 30 PCs, resolution 0.3.
  • Final MG-selected object remains PFlog with cell-cycle genes eligible as HVGs, 20 PCs, resolution 0.5.
  • Tables must match exactly. Figures must remain visually equivalent; device metadata and antialiasing may differ.

Frozen and reporting invariants

  • Do not regenerate any existing object or figure through Leiden clustering during restructuring.
  • Use existing frozen UMAP, clustree, QC, HVG, PCA, and clustering artifacts as-is.
  • sc_analysis.html must render the same visible prose, captions, figures, ordering, and values.
  • sc_analysis.qmd may change only in hidden setup, data loading, and paths.
  • Leave all existing artifacts in place; restructuring is code cleanup, not output cleanup.
Freeze boundary: The implemented candidate is grounded in checksummed RDS files listed by FROZEN_OBJECTS.tsv. Phase 01 is maintenance-only and is not called by routine publication commands. Existing frozen figures remain the comparison baseline.

2. Implemented execution shape

Implemented interface: just run [overwrite] executes phases 02–04 and renders the notebook. The separate just figures, just markers, and just de commands run one publication phase; just regenerate-frozen is deliberate maintenance, requires writable frozen-object directories, and refuses read-only directories.

3. Four scripts, each matching a scientific phase

01 · Regenerate frozen cohort

The maintenance phase is a top-to-bottom workflow for counts, QC, four preprocessing branches, source and MG grids, summaries, and writes.

  • One parameters block owns input directories, QC thresholds, two normalizations, CC inclusion/exclusion, PCs 20/30/50, resolutions 0.3/0.5/0.8, and chosen columns.
  • Ordinary nested loops keep the workflow RStudio-friendly.
  • Writes preserve the established object names, metadata columns, tables, and frozen-stage figures.

02 · Publication figures

The figure phase loads the final source, final MG-selected object, and CC-filtered MG sensitivity object once each.

  • Set each chosen identity column once.
  • Create marker heatmaps, feature/condition/cluster UMAPs, module/p27 heatmaps, and abundance summaries in ordered sections.
  • Use the checked temporary-regular-file mirror sequence for notebook inputs.

03 · Marker analysis

Retain FindAllMarkers as its own descriptive phase because it has a distinct interpretation and output contract.

  • Fixed MG object, assay, layers, thresholds, and chosen cluster column.
  • Encode the selected no-merge analysis directly.
  • Keep marker tables, identity summary, top-five ranking, and dotplot unchanged. These outputs do not feed phase 04.

04 · DE and enrichment

Keep the statistical endpoint separate and readable.

  • Ordered sections: pseudobulk construction, design, DESeq2, shrinkage, paired sensitivity, curated marker overlap, ORA, GSEA, Bayesian enrichment, plots.
  • Keep the Mouse × Condition statistical unit and exact models.
  • Rebuild marker overlap from package marker data plus Cdkn1b, independently of phase 03.

Notebook · report only

The notebook remains a consumer, not an orchestrator.

  • No scientific computation migrates into QMD chunks.
  • Visible prose and captions stay locked.
  • Only setup and path wiring may change.
  • Rendering remains the final presentation step.

Justfile · five obvious commands

  • just run
  • just figures
  • just markers
  • just de
  • just regenerate-frozen

Keep load, document, readme, format, and lint as maintenance commands. The only execution option is the run-level overwrite boolean.

6. Validation: keep only checks that protect meaning

Keep

  • SHA-256, cell-count, and chosen-column matches for all three downstream frozen inputs.
  • Mouse, Condition, and Mouse × Condition sample identity before pseudobulk or exact randomization analyses.
  • Raw integer counts available for DESeq2 aggregation.
  • Control and E-Stim labels both present and contrast direction explicit.
  • Paired-sensitivity feasibility stated explicitly when paired mice are absent.
  • Publication outputs protected from accidental overwrite unless the fixed command requests it.
  • Notebook mirror destinations reject symlinks and temporary copies pass hash and dimension checks.

Remove

  • Generic checks that merely duplicate the next Seurat, plotting, or file operation's clear error.
  • Generic scalar/string/flag parsers and unknown-argument handling.
  • Checkpoint logs, drop-ledger infrastructure, and broad framework instrumentation.
  • Repeated output path normalization, branch-tag parsing, filename translators, and namespace get() calls.
  • Validation of choices that are fixed constants in the publication pipeline.

Rule: add an explicit check only when invalid input could complete successfully with a misleading scientific result, reverse an interpretation, or fail later with a genuinely opaque warning/error.

7. Readability rules for the implementation

  1. Top-to-bottom scripts. Purpose, inputs/outputs, packages, one parameters block, analysis sections, then writes.
  2. Scientific names, not framework names. Use source_cluster, mg_cluster, mouse_condition, and pseudobulk_counts.
  3. Standard calls stay visible. Write Seurat, plotting, file, DESeq2, and enrichment calls directly.
  4. One object load per phase. Set chosen identities once; do not rediscover branches from filenames.
  5. Safe notebook mirrors. Reject symlinks, copy to a temporary regular sibling, verify hash and dimensions, then atomically replace the regular destination.

9. Definition of done

  • Exactly four executable phase scripts plus the notebook.
  • The R package contains only four focused modules that protect nonstandard scientific logic or meaningful reuse.
  • Routine execution has no normalization, PC, resolution, arbitrary-input, merge-map, or output-directory options.
  • Every scientific operation and parameter has an explicit source-to-replacement ledger.
  • All three frozen downstream inputs remain byte-identical and read-only.
  • No frozen-stage figure is regenerated by routine publication commands.
  • Downstream tables match exactly and figures are visually equivalent.
  • The rendered notebook has the same visible content.
  • There are no compatibility aliases, duplicate execution paths, or broad framework checks.
  • A computational biologist can read phases 01–04 in order and understand the complete analysis without first learning a custom pipeline framework.
Core decision: optimize for the reader of this one analysis, not for hypothetical callers. Keep scientific complexity visible; remove software-framework complexity that does not protect a scientific invariant.
