# Writing Conventions

Standards for reports, documents, and writeups in a mycelium-enabled repository.

## Default Format: LaTeX

LaTeX is the default for formal reports. Use the provided `templates/report-template.tex` as the starting point.

For quick informal summaries, Markdown in the analysis documentation file (UPPER_SNAKE_CASE.md) is fine. Reserve LaTeX for reports meant for sharing or publication.

## Document Structure

Standard report sections (in order):

1. **Title and authors**
2. **Abstract** â€” One paragraph summarizing the question, approach, key findings, and implications
3. **Introduction** â€” Context, motivation, specific question addressed
4. **Methods** â€” Data sources, processing steps, statistical methods, software used
5. **Results** â€” Findings with figures and tables, no interpretation
6. **Discussion** â€” Interpretation, limitations, comparison with prior work, next steps
7. **References**
8. **Supplementary material** (if needed)

## Figure Conventions

- **Vector formats preferred**: PDF for LaTeX, SVG for web. Use PNG only when vector is impractical.
- **Resolution**: Minimum 300 DPI for raster images.
- **Consistent styling**: Use a project-wide matplotlib/seaborn style. Set it once in a config file or shared header.
- **Labels**: Every axis must have a label with units. Font size must be readable at the printed/displayed size.
- **Color**: Use colorblind-friendly palettes (e.g., `Set2`, `colorblind` from seaborn). Avoid red-green as the only distinguishing feature.
- **Legends**: Place inside the figure when space allows; outside otherwise.
- **File naming**: Descriptive names â€” `volcano_plot_treatment_vs_control.pdf`, not `fig1.pdf`. LaTeX assigns figure numbers.

## Table Conventions

- Use `booktabs` style in LaTeX (no vertical lines, light horizontal rules)
- Include units in column headers
- Round numbers consistently (typically 2-3 significant figures for statistics)
- Large tables go in supplementary material with a summary in the main text

## Citations and References

- Use BibTeX for citation management
- Store the `.bib` file in the analysis `reports/` directory or a shared `references/` directory
- Prefer DOIs for stable references
- Cite software and packages with their recommended citations

```latex
% In the .tex file
\bibliography{references}
\bibliographystyle{plain}
```

## Referencing Repo Artifacts

When referencing analyses, datasets, or algorithms within the repo:

- Use the manifest name as the canonical identifier
- Example: "We used the `deseq2-wrapper` algorithm (see `algorithms/deseq2-wrapper/`) applied to the `rnaseq-batch-1` dataset (see `data/DATA_MANIFEST.md`)."
- For figures generated by an analysis: reference the analysis name and output path

## Using the Report Template

The `templates/report-template.tex` provides:
- Standard preamble with common packages
- Pre-configured `booktabs`, `graphicx`, `hyperref`
- Placeholder sections matching the structure above
- Comment blocks explaining what goes where

To use:
1. Copy to `analysis/[name]/reports/[name]-report.tex`
2. Fill in the placeholders
3. Add figures from `analysis/[name]/outputs/`
4. Compile with `pdflatex` + `bibtex` (or `latexmk`)

## Style Guide

- Write in past tense for methods and results ("We performed...", "The analysis showed...")
- Use active voice when possible
- Define abbreviations on first use
- Be precise about statistical claims â€” "associated with" not "caused by" (unless causation is established)
- Keep paragraphs focused â€” one idea per paragraph
