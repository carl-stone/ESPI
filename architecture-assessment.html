1. End-to-end shape

One routine pipeline and one maintenance boundary. just run [overwrite] executes phases 02–04 and renders the notebook. just regenerate-frozen runs phase 01 only and requires writable seurat_objects/input and seurat_objects/current directories.

2. Fixed publication choices

Selected source branch

PFlog with cell-cycle genes retained as HVG candidates, 30 PCs, resolution 0.3. The contract expects 4,146 cells and cluster column cluster_pflog_no_filter_cc_dims30_res0.3.

Selected MG branches

The primary MG-selected object uses PFlog, cell-cycle genes retained, 20 PCs, and resolution 0.5 with 3,456 cells. A same-size cell-cycle-HVG-filtered object remains the sensitivity branch.

“No filter CC” means cell-cycle genes remain eligible as highly variable genes; it does not mean cells were filtered by condition or cell-cycle state.

4. Validation and pressure points

Meaning-protecting checks

  • SHA-256, cell count, and chosen cluster column for frozen inputs.
  • Explicit Mouse × Condition identities and full-rank DE design.
  • Raw integer counts before pseudobulk DESeq2.
  • Explicit condition labels and contrast direction.
  • Overwrite refusal for existing publication outputs.
  • Symlink rejection plus hash and dimension checks for notebook figure mirrors.

Remaining maintenance risks

  • No analysis-focused automated test suite or CI workflow.
  • The phase scripts and publication-analysis module remain large.
  • Committed HTML and figure artifacts can drift from their sources unless regenerated.
  • The external data root is machine-specific and must be configured locally when the default Box path is unavailable.

5. Source-of-truth hierarchy

  1. Execution: the four phase scripts and justfile.
  2. Shared scientific implementation: the four modules in R/ and tracked package data.
  3. Scientific narrative: notebook/sc_analysis.qmd and CONTEXT.md.
  4. Operational documentation: AGENTS.md, README.Rmd, environment instructions, and manifests.
  5. Generated artifacts: README.md, Rd files, rendered notebook HTML, and regular-file notebook figure mirrors.
