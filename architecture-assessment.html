ESPI analysis architecture

Current repository shape, traced from executable scripts and package helpers. The analysis is a small R package wrapped around a procedural Seurat pipeline, with large objects and most generated results stored outside Git.

1. End-to-end shape

Two operational pipelines coexist. just regenerate-frozen runs counts → QC → branch grids → MG selection. Routine just run starts from frozen clustered MG-selected RDS files and runs summaries, figures, markers, DE, enrichment, notebook rendering, and tripwires. The default command therefore does not rebuild scripts 01–07.

“No filter CC” means cell-cycle genes remain eligible as HVGs; it does not remove cells.

4. Structural pressure points

Where complexity concentrates

  • run-pipeline.R duplicates stage defaults, path rules, argument assembly, validation, and frozen/current policy.
  • cluster-grid-summary.R combines loading, parsing, metrics, tables, and plots.
  • Large procedural endpoints: 12-run-mg-de.R (1,609 lines), 11-find-mg-markers.R (993), and 09-plot-mg-figures.R (852).
  • CLI parsing, branch-tag construction, cluster sorting, and PNG/PDF writing recur in multiple files.

Where the current scope exceeds the final need

  • Legacy, counts-QC, and arbitrary-RDS input modes remain supported.
  • Candidate-grid machinery is exposed through scripts, just recipes, package exports, and the orchestrator.
  • Hyphenated filenames and Seurat-safe underscore tags require translation logic.
  • Many parameter-encoded figures and objects coexist, so stale alternatives are easy to retain.
  • The pipeline continues past the requested two chosen Seurat objects into MG selection and manuscript analyses.

Documentation drift observed: algorithms/ALGORITHM_MANIFEST.md names scripts/qc-filtering.R, while the executable file is scripts/02-qc-filtering.R. The assessment therefore treats source paths and orchestration as authoritative.

5. Source-of-truth hierarchy

  1. Execution: numbered scripts, run-pipeline.R, and justfile.
  2. Shared implementation: R/ package modules and tracked package data.
  3. Scientific narrative: notebook/sc_analysis.qmd and CONTEXT.md.
  4. Operational history: manifests, .living/decisions.md, and .living/learnings.md.
  5. Derived artifacts: external Box outputs, notebook symlinks, rendered HTML, generated README, NAMESPACE, and Rd files.

Assessment basis: all executable scripts, all R/ modules, orchestration recipes, package metadata, notebook source, manifests, Mycelium index, generated-artifact conventions, and source line counts. Generated plot contents and large external RDS objects were not re-run or re-audited.
