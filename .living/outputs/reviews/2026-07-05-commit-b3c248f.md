##### F2. Abundance direction colors have a second source of truth

`R/cluster-abundance.R:245-250`
```r
    ggplot2::scale_fill_manual(
      values = c(
        "Enriched in E-Stim" = "#e31a8c",
        "Depleted in E-Stim" = "#2166ac",
        "Not significant" = "grey75"
      ),
```
**Why it matters here**: The same pink/blue endpoints already encode low/high expression elsewhere in ESPI through `palette_dotplot_pair`. Repeating raw hex values is not wrong today, but a future palette change could make abundance direction colors drift from the rest of the manuscript.
**Fix**: Define a named abundance-direction palette in `R/themes.R`, or derive the enriched/depleted colors from `palette_dotplot_pair` in this helper.

##### F3. Exported abundance plot labels are MG-selected-specific

`R/cluster-abundance.R:256-260`
```r
    ggplot2::labs(
      x = "MG-selected cluster",
      y = "CLR log2 enrichment (E-Stim vs control)",
      fill = NULL,
      subtitle = "Pooled cell-level Fisher/CLR summary; descriptive relative to Mouse x Condition DE unit."
```
**Why it matters here**: `compute_cluster_abundance()` accepts configurable condition labels, but the exported plotting helper always labels the contrast as E-Stim vs control and the x-axis as MG-selected. Reusing it outside this exact branch would silently produce biologically wrong labels.
**Fix**: Either keep the plotting helper internal/MG-specific by name, or add `x_label` and `contrast_label` arguments passed from the same analysis parameters used for computation.
